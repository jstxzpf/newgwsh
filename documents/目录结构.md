# 国家统计局泰兴调查队公文处理系统 V3.0 - 项目文件结构

依据核心设计方案与软硬件隔离约束，本项目采取标准的前后端分离架构设计，并基于 Docker Compose 统一部署。建议的代码库结构如下，以确保后续开发、维护和信创环境部署时的职责清晰与代码高内聚。

## 根目录 (Root)
整个项目的顶层目录，主要包含容器编排与环境配置文件。
```text
newgwsh/
├── docker-compose.yml              # 生产环境容器编排 (网络隔离、无端口暴露)
├── docker-compose.dev.yml          # 开发环境容器编排 (使用 bind mount，暴露调试端口)
├── .env.example                    # 环境变量配置模板
├── .gitignore                      # Git 忽略规则
├── README.md                       # 项目说明与快速启动指南
│
├── frontend/                       # 前端源码目录 (React)
└── backend/                        # 后端源码目录 (FastAPI + Celery)
```

---

## 1. 前端结构 (`frontend/`)
采用 React 19 + TypeScript + Vite 搭建，使用 Ant Design v6 主题与 Zustand 状态管理。
```text
frontend/
├── package.json                    # 前端依赖配置
├── vite.config.ts                  # Vite 构建与代理配置
├── tsconfig.json                   # TypeScript 配置
├── index.html                      # HTML 模板
├── .eslintrc.cjs                   # ESLint 规范配置
├── public/                         # 静态资源存放处 (如 favicon, 默认占位图等)
│
└── src/
    ├── main.tsx                    # React 挂载入口
    ├── App.tsx                     # 根组件 (包裹全局路由、水印组件)
    ├── routes/                     # 路由拓扑定义 (如 /dashboard, /workspace/:doc_id 等)
    │
    ├── theme/                      # 视觉主题防御链配置
    │   └── antd-theme.ts           # Ant Design v6 定制 Token (政务蓝、方正仿宋字体链)
    │
    ├── components/                 # 可复用全局组件
    │   ├── AntiLeakWatermark/      # 全局防泄密水印组件
    │   ├── GlobalTaskWatcher/      # 隐身组件：统一管理全局 SSE 任务进度监听
    │   ├── LockConflictBanner/     # 隔离冲突死锁警戒仪横幅
    │   ├── SnapshotRecoveryDrawer/ # 历史快照灾备抽屉
    │   └── EditorA4Paper/          # A4 拟物化引擎画板包裹器
    │
    ├── pages/                      # 页面级视图层
    │   ├── Login/                  # 登录页
    │   ├── Dashboard/              # 个人工作台 (任务聚焦板)
    │   ├── Workspace/              # 沉浸式公文工作区 (左右双栏、DIFF 引擎)
    │   ├── Knowledge/              # 统计知识资产库视图
    │   ├── Approvals/              # 科长签批管控台
    │   ├── Chat/                   # HRAG 穿透式智能问答台
    │   └── Settings/               # 系统中枢设置 (用户、参数、大盘、审计等)
    │
    ├── stores/                     # Zustand 状态机 (统一数据源)
    │   ├── useAuthStore.ts         # 鉴权与用户信息
    │   ├── useEditorStore.ts       # 工作区草稿、DIFF对比态、快照容灾管理
    │   └── useTaskStore.ts         # SSE 异步任务进度追踪管理
    │
    ├── services/                   # API 接口统一封装与 Axios 拦截器
    │   ├── apiClient.ts            # Axios 实例 (JWT 拦截、401 踢出处理、请求响应日志)
    │   └── api/                    # 按业务域划分的 API 请求函数集 (auth, docs, locks 等)
    │
    ├── hooks/                      # 自定义 React Hooks
    │   ├── useLockGuard.ts         # Web Worker 抢锁与高精度心跳续期
    │   └── useEditorNotifications.ts # 个人全局事件监听 (如锁被踢出、审批被驳回)
    │
    ├── utils/                      # 工具函数集
    │   ├── wordCount.ts            # Markdown 纯文本字数统计引擎
    │   └── format.ts               # 日期与常用格式化
    │
    └── types/                      # 全局 TypeScript 接口与类型定义
```

## 2. 后端结构 (`backend/`)
采用 FastAPI 作为高速网关，Celery 分离长耗时任务，SQLAlchemy 2.0 管理异步/同步会话。
```text
backend/
├── pyproject.toml                  # 依赖管理 (建议使用 poetry 或 uv)
├── requirements.txt                # Pip 依赖表
├── Dockerfile                      # 后端镜像多阶段构建文件
├── alembic.ini                     # 数据库版本控制配置
├── start.sh                        # 容器启动脚本 (自动执行 Alembic 升级并启动 Uvicorn)
│
├── alembic/                        # 数据库迁移脚本存放目录
│
└── app/                            # 后端源码根目录
    ├── main.py                     # FastAPI 应用入口与中间件注册
    ├── celery_app.py               # Celery Broker 与 Worker 实例化配置
    │
    ├── core/                       # 系统级核心基建层
    │   ├── config.py               # Pydantic 动态环境变量与参数
    │   ├── database.py             # AsyncSessionLocal (给FastAPI) 与 SyncSessionLocal (给Celery)
    │   ├── security.py             # Argon2id 密码哈希验证与 JWT Token 签发解析
    │   ├── sip.py                  # NFKC 归一化与 HMAC-SHA256 防篡改存证核心算法
    │   └── exceptions.py           # 全局定制化异常处理与 HTTP Code 映射
    │
    ├── api/                        # 路由与控制器层
    │   ├── dependencies.py         # FastAPI 依赖注入 (验证 Token, 获取 DB 会话)
    │   └── v1/                     # 接口版本
    │       ├── auth.py             # 登录鉴权、Token无感刷新
    │       ├── documents.py        # 公文生命周期控制、草稿自动保存
    │       ├── approval.py         # 审阅签批流、回退流
    │       ├── tasks.py            # AI 推理/排版任务触发网关
    │       ├── sse.py              # SSE 票据核发与 Server-Sent Events 流通道
    │       ├── locks.py            # Redlock 分布式并发悲观锁申请与释放
    │       ├── kb_admin.py         # 知识库层级化上传与删除管理
    │       ├── chat.py             # HRAG 智能穿透对话路由
    │       ├── audit.py            # 审计安全查询路由
    │       └── sys.py              # 全局系统参数配置、账户管控、指标探针
    │
    ├── models/                     # SQLAlchemy 实体关系模型 (ORM)
    │   ├── enums.py                # 全局业务枚举 (DocumentStatus, DataSecurityLevel 等)
    │   ├── user.py                 # system_users, departments, users_sessions
    │   ├── document.py             # documents, document_types, exemplar_documents, snapshots
    │   ├── knowledge.py            # knowledge_base_hierarchy, knowledge_chunks, physical_files
    │   └── system.py               # async_tasks, nbs_workflow_audit, approval_logs, system_config
    │
    ├── schemas/                    # Pydantic 数据契约 (请求体与响应体验证)
    │   ├── request/                # DTO 接收数据模型
    │   └── response/               # DTO 返回数据模型
    │
    ├── services/                   # 跨表核心业务逻辑与原子性事务控制
    │   ├── document_service.py     # 复杂公文流转 (如：回退状态并原子抢锁)
    │   ├── rrag_service.py         # HNSW向量召回 + BM25文本召回 + RRF 融合处理
    │   └── chunk_service.py        # 软删除级联处理逻辑与索引清理
    │
    ├── tasks/                      # Celery 异步任务
    │   ├── worker.py               # 异步任务接收器 (包含 POLISH / FORMAT / PARSE 任务及跨进程复核逻辑)
    │   ├── ast_chunker.py          # markdown-it-py 语义切片、表格 RAG 行提取
    │   └── doc_formatter.py        # GB/T 9704-2023 国标 Word 排版输出与排重检测
    │
    ├── prompts/                    # 外置提示词中心 (通过 Volume 挂载，支持热加载)
    │   ├── prompt_loader.py        # 提示词内存加载器与刷新钩子
    │   ├── system_chat.txt         # HRAG 系统防幻觉提示词
    │   ├── system_polish.txt       # AI 公文润色指令模板
    │   └── vocab_blacklist.txt     # 政务词汇违禁词黑名单表
    │
    ├── data/                       # 宿主机挂载的持久化物理目录
    │   ├── uploads/                # 知识库上传的原始文件、图片存储
    │   └── outputs/                # 生成的 .docx 正式文档产出目录
    │
    └── scripts/                    # 运维与辅助初始化工具
        ├── seed_data.py            # 初始化超级管理员、科室字典与配置种子
        └── check_users.py          # 查询核验用户权限与状态
```
