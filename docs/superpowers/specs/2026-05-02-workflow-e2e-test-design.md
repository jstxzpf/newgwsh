# E2E 测试：完整业务流与 AI 聊天设计规格说明

**日期：** 2026-05-02
**主题：** 编写模拟 AI 聊天和完整业务流的 Playwright E2E 测试。

## 1. 目标
实现一个端到端的自动化测试脚本，模拟用户登录、进入知识库、进入智能助手并发送消息获取 AI 响应。

## 2. 场景描述
1. 用户访问 `/login` 页面。
2. 输入管理员凭据（admin/admin123）并登录。
3. 验证跳转至 `dashboard`。
4. 点击侧边栏的“知识库”链接。
5. 验证进入知识库页面（检查页面标题或特定元素）。
6. 点击侧边栏的“智能助手”链接。
7. 在聊天框中输入“你好，gemma”并发送。
8. 验证 AI 响应出现（检查选择器 `.chat-message-ai`）。

## 3. 技术实现方案

### 3.1 前端调整 (为了使测试通过)
由于当前前端代码中缺少测试所需的选择器和标题，需进行以下微调：
- **`MainLayout.tsx`**: 确保导航项可以被 `text=知识库` 和 `text=智能助手` 识别。
- **`KnowledgeBase.tsx`**: 添加 `h1` 标签内容为“知识库管理”。
- **`Chat.tsx`**: 为 AI 消息容器添加 `.chat-message-ai` 类名。

### 3.2 测试脚本
- **文件路径**: `e2e-tests/tests/workflow.spec.ts`
- **工具**: Playwright

## 4. 验收标准
- 运行 `npx playwright test workflow.spec.ts` 能够全部通过（在具备环境的情况下）。
- 代码符合项目规范，使用 TypeScript 编写。
- 更改已 commit 到 git。
